### Phase 9 — `npm audit` triage + final sweep (complete)

- **`npm audit` findings from session 1 resolved.** All five moderate-severity vulnerabilities chained back to one root cause: `yaml@2.0.0–2.8.2` has a stack-overflow via deeply nested YAML, pulled in transitively through `yaml-language-server` → `volar-service-yaml` → `@astrojs/language-server` → `@astrojs/check`. The default `npm audit fix --force` would have downgraded `@astrojs/check` from `^0.9.8` to `0.9.2` (breaking). Instead, added an `overrides` block to `package.json` pinning `yaml` to `^2.8.3` (one patch above the vulnerable range). `npm install` removed the nested vulnerable copy, `npm audit` now reports `0 vulnerabilities` across 461 packages, and the downstream tooling (`@astrojs/check`, language server, `prettier-plugin-astro`) still works — verified by `npm run build`, `npm run check`, `npm run lint`.
- **Rationale for overrides over audit-fix-force:** the exposure for a static personal site was zero to begin with (yaml is only parsed at build time, and only from Ray's own content), but a clean audit is worth the one-line override so future sessions don't see the warning and waste cycles re-investigating it. The override is documented in this session log so future sessions know it's intentional and can remove it once upstream `yaml-language-server` bumps its transitive dep.
